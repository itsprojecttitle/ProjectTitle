## Usage

* This template uses tailwind css every tailwind class are prefixed with `tw-`, to help differentiate
  between tailwind classes and other classes  

During development add the following to head tag

```html
<link rel="stylesheet" href="tailwind-runtime.css"><!--replace with path to your tailwind runtime-->
```
During production use

```html
<link rel="stylesheet" href="tailwind-build.css"><!--replace with path to your tailwind build-->
```

To start Tailwind during development use
```html
npm run start:tailwind
```

To create a build file use
```html
npm run build:tailwind
```

## To Start Local Development:

After applying the above changes and ensuring all files are saved:

1.  **In one terminal:** Navigate to `/Users/samuelapata/Desktop/Projecttitle` and run:
    npm run start:tailwind

    This will compile your CSS and watch for changes.

2.  **In a *separate* terminal:** Navigate to `/Users/samuelapata/Desktop/Projecttitle` and run:
    http-server

    This will serve your website, likely at `http://localhost:8080`.

3.  **Open your browser** to the address provided by `http-server` (e.g., `http://localhost:8080`).

### 3. Create `postcss.config.js`

Create this file in the root of the project to enable PostCSS processing for Tailwind CSS and Autoprefixer.

**`postcss.config.js` content:**
  module.exports = {
    plugins: {
      tailwindcss: {},
      autoprefixer: {},
    },
  };
